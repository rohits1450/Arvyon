## Key Learnings & Design Decisions

### ZK Circuit Design
- **Range proof approach**: Private inputs (actionValue, policyMin, policyMax) with public compliance output
- **Why range proof**: Minimal constraints (~3), fast verification, perfect for policy bounds
- **Why BN254**: Standard for EVM, Solidity verifier compatibility, audited by community

### Executor Integration
- **Mock proof→ Real proof**: Signature accepts uint[2], uint[2][2], uint[2] matching Groth16 commitment structure
- **Proof path**: TIS JSON → SnarkJS → proof.json → Executor.verifyProof() → PDRLogger.logDecision()
- **Privacy guarantee**: Only compliance Boolean public; action values remain private

### Agent Architecture
- **3-node model**: Observe (state fetch) → Decide (LLM) → Intent (TIS + proof)
- **Loop-friendly**: Each node returns modified state for next iteration
- **LLM-ready**: Mock decisions work; real decisions via Anthropic API with one import change
